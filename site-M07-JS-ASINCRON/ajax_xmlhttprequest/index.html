    2. AJAX amb XMLHttpRequest#

    AJAX (Asynchronous JavaScript and XML) és una tècnica que permet fer peticions al servidor de manera asíncrona, evitant recarregar tota la pàgina. Utilitza principalment l'objecte XMLHttpRequest, tot i que avui dia també s'utilitza molt fetch().

    📊 Flux d’una petició AJAX amb XMLHttpRequest#

    sequenceDiagram
    participant JS
    participant Server

    JS->>Server: open() + send()
    Server-->>JS: Retorna resposta (XML/JSON)
    JS->>JS: onreadystatechange → callback
    JS->>JS: Gestiona i mostra dades

    🧱 Components bàsics de XMLHttpRequest#

    🔧 Creació d'una petició GET#

    let xmlHttp = new XMLHttpRequest();
let url = "valida.php?nom=cristian&edat=8";

xmlHttp.open("GET", url, true); // true indica que és asíncrona
xmlHttp.onreadystatechange = function () {
  if (xmlHttp.readyState === 4 && xmlHttp.status === 200) {
    console.log(xmlHttp.responseText);
    gestionaResposta(xmlHttp);
  }
};
xmlHttp.send(null);

    🔁 Estats de readyState#

    Estat Descripció
    0 No iniciada
    1 Connexió oberta
    2 Headers rebuts
    3 Rebent dades
    4 Petició completada

    📤 Petició POST amb XMLHttpRequest#

    let xmlHttp = new XMLHttpRequest();
let url = "valida.php";

xmlHttp.open("POST", url, true);
xmlHttp.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
xmlHttp.onreadystatechange = function () {
  if (xmlHttp.readyState === 4 && xmlHttp.status === 200) {
    console.log(xmlHttp.responseText);
    gestionaResposta(xmlHttp);
  }
};
xmlHttp.send("nom=cristian&edat=88");

    📥 Exemple de resposta en XML des de PHP#

    <?php
$name = $_GET['nom'];
header("Content-type: text/xml");
echo '<?xml version="1.0" encoding="utf-8"?>';
echo "<nom><![CDATA[$name]]></nom>";
?>

    🔍 Tractament de la resposta XML#

    function repResposta(xmlHttp) {
  let resp = xmlHttp.responseXML;
  let listResp = resp.getElementsByTagName("nom");
  for (let k = 0; k < listResp.length; k++) {
    let result = listResp[k].childNodes[0].nodeValue;
    alert(result);
  }
}

    🧪 Exemple de resposta JSON des de PHP#

    <?php
$name = $_GET['nom'];
$resposta = '{"nom":"' . $name . '", "noms":["Cristian","Pedro"], "info":{"noms":["nom1","nom2"]}}';
echo $resposta;
?>

    🔎 Tractament de la resposta JSON#

    function repRespostaJSON(xmlHttp) {
  if (xmlHttp.status === 200) {
    let resp = xmlHttp.responseText;
    let respJSON = JSON.parse(resp);
    let txt = respJSON.nom;
    let nom1 = respJSON.noms[0];
    let nom2 = respJSON.noms[1];
    console.log(txt, nom1, nom2);
  }
}

    📦 Enviament de fitxers amb FormData#

    <input type="file" id="arxius">
<input type="button" onclick="sendFile();">

    function sendFile() {
  let control = document.getElementById("arxius");
  let arxiu = control.files[0];
  let form = new FormData();
  form.append("arxiu", arxiu);

  let xmlHttp = new XMLHttpRequest();
  xmlHttp.open("POST", "ajax.php", true);
  xmlHttp.send(form);
}

    🧩 Casos d’ús reals amb XMLHttpRequest#

    📄 Formularis de registre#

    Validació d'usuari disponible en temps real sense recarregar la pàgina.

    🗂️ Gestió d'arxius#

    Enviament de documents, imatges o fitxers via formulari amb FormData.

    🎮 Aplicacions interactives#

    Jocs en línia que intercanvien dades amb el servidor de manera contínua (p. ex. puntuació en temps real).

    📈 Dashboard i analítica#

    Actualització de gràfiques o dades estadístiques sense recarregar la pàgina sencerament.

    🔄 Resum de conceptes clau#

    Concepte Explicació
    readyState Estat de la petició (de 0 a 4)
    status Codi HTTP de resposta del servidor (ex. 200 = OK, 404 = Not Found)
    responseText Contingut de la resposta del servidor en format text (ex: "hola ALF").
    responseXML Contingut de la resposta en format XML (si el servidor respon amb XML).
    JSON.parse(...) Converteix una resposta en format JSON (text) a un objecte JavaScript.
    setRequestHeader Afegeix capçaleres HTTP a la petició, útil en mètodes com POST o PUT.
    open(metode, url, async) Defineix el mètode (GET/POST), la URL i si és una petició asíncrona.
    send(dades) Envia la petició al servidor; pot portar dades en cas de POST.
    onreadystatechange Funció que s'executa cada vegada que canvia l'estat de readyState.
    xmlHttp Objecte creat amb new XMLHttpRequest() per fer la petició.
    console.log(...) Serveix per mostrar informació a la consola (molt útil per depurar).
    true a open(...) Indica que la petició és asíncrona (no bloqueja l'execució del codi).
    false a open(...) Indica petició síncrona (desaconsellada perquè bloqueja el navegador).
    Content-Type Capçalera que indica el tipus de contingut enviat (ex: application/json).
    XMLHttpRequest Objecte integrat de JavaScript per fer peticions HTTP al servidor.
    FormData Objecte per construir dades a enviar via POST, inclosos fitxers

    ❗️Errors habituals amb XMLHttpRequest#

    ❌ No establir el mètode correctament#

    xmlHttp.open("GET", url); // Falta el tercer paràmetre (true/false)
    ➡ Solució: Afegeix true per fer-la asíncrona: xmlHttp.open("GET", url, true);

    ❌ No afegir capçalera en peticions POST#

    // Sense setRequestHeader() en POST
xmlHttp.open("POST", url, true);
xmlHttp.send("param1=valor");
    ➡ Solució: xmlHttp.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");

    ❌ Gestionar malament l'estat readyState#

    if (xmlHttp.readyState === 200) { ... } // ERROR
    ➡ Solució: Comprova primer que readyState === 4 i després status === 200

    ❌ Errors de CORS#

    Si el servidor no permet peticions des d’un domini diferent, pot fallar la petició. ➡ Solució: El servidor ha d’afegir l'encapçalament Access-Control-Allow-Origin: *

    ❌ No controlar errors de xarxa#

    xmlHttp.onreadystatechange = function () {
  if (xmlHttp.readyState === 4) {
    // No comprova si xmlHttp.status !== 200
  }
};
    ➡ Solució: Comprova sempre xmlHttp.status i gestiona casos com 404 o 500
