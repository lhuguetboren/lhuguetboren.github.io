    3. AJAX amb Fetch#

    La funció fetch() és una alternativa moderna a XMLHttpRequest per fer peticions HTTP. És més neta i senzilla d’utilitzar, basada en promeses (Promise) i compatible amb async/await.

    ⚖️ Comparativa: XMLHttpRequest vs Fetch#

    Característica XMLHttpRequest fetch()
    API basada en Esdeveniments (onreadystatechange) Promeses (.then(), .catch())
    Sintaxi Verbosa i complexa Clara i moderna
    Tractament d'errors S’ha de gestionar amb estats i codis manualment Es pot gestionar amb catch()
    Suport per JSON Manual (JSON.parse(), JSON.stringify()) Integrat (response.json())
    Càrrega de fitxers Mitjançant FormData + send() Mitjançant FormData + body
    Cancel·lació de peticions No suportada per defecte Amb AbortController
    Suport en navegadors antics Molt compatible Millor en navegadors moderns

    Una promesa (Promise) representa una operació asíncrona que encara no ha finalitzat, però que ho farà en el futur. Pot estar en un d’aquests tres estats:

    • Pending: en espera (pendent)
    • Fulfilled: resolta amb èxit
    • Rejected: rebutjada per un error

    🔄 Flux de vida d’una Promise#

    graph TD
    A["Promise creada (pending)"] -->|Operació completada| B["resolve() → fulfilled"]
    A -->|Error o fallada| C["reject() → rejected"]
    B --> D["then() → gestiona èxit"]
    C --> E["catch() → gestiona error"]

    📊 Flux de petició amb Fetch#

    sequenceDiagram
    participant JS
    participant Server

    JS->>Server: fetch(url, config)
    Server-->>JS: Retorna resposta (JSON/XML/Text)
    JS->>JS: then() → processar resposta

    Les promeses permeten escriure codi més net i estructurat per gestionar operacions asíncrones com peticions a servidors, temporitzadors, lectura de fitxers, etc.

    🚀 Exemple de petició GET amb Fetch#

    let configFetch = {
  method: "GET",
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  }
};

fetch("ejemplo.php?n1=2&n2=3", configFetch)
  .then(response => response.json())
  .then(objetoJSON => {
    console.log(objetoJSON);
  })
  .catch(err => console.log("Error en la petició", err));

    📤 Exemple de petició POST amb Fetch#

    let configFetch = {
  method: "POST",
  body: "n1=2&n2=3",
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  }
};

fetch("ejemplo.php", configFetch)
  .then(response => response.json())
  .then(objetoJSON => {
    console.log(objetoJSON);
  });

    🧰 Funcions útils de la resposta#

    Aquí tens la taula ampliada amb més funcions i propietats útils de la resposta (Response) en fetch:

    Funció / Propietat Descripció
    response.ok Indica si la resposta té un codi d'estat entre 200 i 299 (èxit).
    response.status Retorna el codi d'estat HTTP (ex: 200, 404, 500...).
    response.statusText Missatge de l'estat HTTP (ex: "OK", "Not Found").
    response.url Retorna la URL final de la resposta (pot ser diferent si hi ha redirecció).
    response.headers Objecte amb totes les capçaleres de la resposta (Headers).
    response.json() Converteix el cos de la resposta a un objecte JSON (promesa).
    response.text() Retorna el cos de la resposta com a text (promesa).
    response.blob() Retorna el cos com a Blob, útil per descarregar arxius.
    response.formData() Retorna el cos com a FormData, habitual en formularis.
    response.arrayBuffer() Retorna el cos com a dades binàries (ArrayBuffer).
    response.redirected Indica si la resposta és resultat d'una redirecció automàtica (true/false).
    response.type Tipus de resposta (basic, cors, opaque, etc.).
    response.bodyUsed Indica si el cos de la resposta ja s'ha llegit (true/false).

    📌 Nota: Les funcions com .json(), .text() i .blob() retornen promeses, perquè l'accés al cos pot ser asíncron.

    📦 Enviament de fitxers amb Fetch i FormData#

    <input type="file" id="arxiu">
<input type="button" onclick="enviarFitxer()">

    function enviarFitxer() {
  let control = document.getElementById("arxiu");
  let form = new FormData();
  form.append("arxiu", control.files[0]);
  form.append("param1", "valor1");
  form.append("param2", "valor2");

  fetch("http://localhost:8089/endpoint", {
    method: "POST",
    body: form
  })
  .then(resp => resp.json())
  .then(respJSON => {
    console.log(respJSON);
  });
}

    ❗️Tractament d’errors amb Fetch#

    fetch() no llença errors automàticament si el servidor retorna un codi com 404 o 500. Cal verificar-ho manualment:

    fetch("endpoint.php")
  .then(response => {
    if (!response.ok) {
      throw new Error("Error HTTP: " + response.status);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error("S'ha produït un error:", error));

    Altres errors habituals: - ❌ Error de sintaxi a fetch (oblit de {} o cometes) - ❌ body mal format (ex. no usar JSON.stringify() per a JSON) - ❌ Capçalera Content-Type no coincident amb el cos - ❌ Servidor sense suport per CORS (error de seguretat en cross-domain)

    💡 Casos d’ús habituals amb Fetch#

    • Consultes d’API REST: recuperació de dades des de serveis externs.
    • ✏️ Formularis de contacte: enviament dinàmic de dades sense reload.
    • 🖼️ Carrega d’imatges i fitxers: gestió de fitxers mitjançant FormData.
    • 📊 Aplicacions SPA: Fetch permet la càrrega dinàmica de contingut en aplicacions d’una sola pàgina.

    📘 Glossari de termes clau per Fetch#

    Terme Definició
    fetch() Funció moderna per fer peticions HTTP en JavaScript
    Promise Objecte que representa un resultat futur (èxit o error)
    async/await Sintaxi moderna per treballar amb promeses de forma més clara
    headers Objecte amb les capçaleres HTTP a enviar amb la petició
    body Contingut que s’envia en una petició POST/PUT
    Content-Type Tipus de dades que s’envien (ex. JSON, form-urlencoded, multipart...)
    response.ok Booleà que indica si la resposta és exitosa (status 200–299)
    response.status Codi HTTP retornat pel servidor (ex. 200, 404)
    response.json() Converteix la resposta a objecte JavaScript (si el contingut és JSON)
    response.text() Llegeix la resposta com a text pla
    FormData Objecte per encapsular fitxers i camps de formulari per POST

    🧪 Alternativa: Axios#

    Axios és una llibreria basada en promeses com fetch, però amb més funcionalitats per defecte:

    npm install axios

    import axios from 'axios';

axios.post("/endpoint", {
  nom: "Maria",
  edat: 25
})
.then(response => {
  console.log(response.data);
})
.catch(error => {
  console.error("Error a la petició", error);
});

    Avantatges de Axios:#

    • Suporta automàticament JSON
    • Gestiona errors HTTP com excepcions
    • Permet cancel·lar peticions
    • Funciona tant al navegador com a Node.js
