    📘 Concepto: Colecciones Time-Series en MongoDB#

    ¿Qué es una colección Time-Series?#

    Una colección time-series está diseñada específicamente para almacenar datos que llegan en secuencia temporal, como los provenientes de sensores IoT.

    MongoDB almacena automáticamente estos datos en "buckets" comprimidos para ahorrar espacio y acelerar consultas por tiempo.

    🏷️ Características clave#

    Propiedad Descripción
    timeField Campo que representa el timestamp (obligatorio)
    metaField Campo con información común al dispositivo o sensor (opcional pero útil)
    granularity Nivel de precisión esperado: "seconds", "minutes" o "hours"
    Optimización interna MongoDB organiza datos en buckets por eficiencia

    📄 Ejemplo de documento#

    🧪 Cómo crear una colección Time-Series en Python#

    from pymongo import MongoClient

client = MongoClient("mongodb://localhost:27017")
db = client["ecm_mongo_db"]

# Solo crear si no existe
if "DeviceData" not in db.list_collection_names():
    db.create_collection(
        "DeviceData",
        timeseries={
            "timeField": "timestamp",         # obligatorio
            "metaField": "metadata",          # opcional pero recomendable
            "granularity": "seconds"          # "seconds", "minutes", "hours"
        }
    )
    print("✅ Colección time-series creada.")

    ¿Por qué usar Time-Series en lugar de una colección normal?#

    Ventaja Normal Time-Series
    Compresión automática ❌ ✅
    Optimización por tiempo ❌ ✅
    Consulta eficiente por fecha ⚠️ ✅
    TTL automático (opcional) ✅ ✅

    🧰 Buenas prácticas#

    • Usa datetime.utcnow() para mantener coherencia de zona horaria.
    • Usa metadata.device_id como clave de agrupación.
    • Crea índices secundarios si haces muchas búsquedas por metadata.variable_type.

    🧱 4. Modelo de Datos (10 min)#

    schemas.py

    from datetime import datetime

class Reading:
    def __init__(self, device_id, temperature, timestamp=None):
        self.timestamp = timestamp or datetime.utcnow()
        self.metadata = {
            "device_id": device_id,
            "variable_type": "temperature"
        }
        self.data = { "temperature": temperature }
        self.device_id = device_id

    La estructura que Mongo espera: - timestamp automático con datetime.utcnow(). - Campos metadata, data, device_id.

    🧠 5. Operaciones CRUD (10 min)#

    models.py

    def add_reading(reading):
    readings_collection.insert_one(vars(reading))

def get_readings(device_id, start_time=None, end_time=None):
    query = {"device_id": device_id}
    if start_time and end_time:
        query["timestamp"] = {"$gte": start_time, "$lte": end_time}
    elif start_time:
        query["timestamp"] = {"$gte": start_time}
    elif end_time:
        query["timestamp"] = {"$lte": end_time}
    return list(readings_collection.find(query))
