    PYTHON + MONGODB#

    Base de Datos en MongoDB#

    Una base de datos en MongoDB es un contenedor que almacena colecciones de documentos JSON. A diferencia de las bases de datos relacionales tradicionales (SQL), MongoDB es NoSQL, lo que significa que no usa tablas, filas ni relaciones estrictas entre entidades. En su lugar, los datos se organizan en estructuras más flexibles denominadas documentos dentro de colecciones.

    Características Claves de una Base de Datos en MongoDB#

    1. Almacenamiento de Documentos JSON/BSON
    2. Cada registro en MongoDB se almacena en un documento JSON (internamente convertido a BSON para optimización).

    3. Un documento puede contener diferentes campos y estructuras, sin necesidad de un esquema fijo.

    4. No usa Esquema Fijo

    5. En bases de datos relacionales (SQL), cada tabla tiene columnas con tipos de datos definidos.

    6. En MongoDB, cada documento dentro de una colección puede tener diferentes campos, lo que brinda mayor flexibilidad.

    7. Escalabilidad Horizontal

    8. MongoDB permite distribuir datos en varios servidores mediante un sistema de sharding, lo que mejora la escalabilidad.

    9. Consultas Rápidas

    10. Utiliza índices y un potente motor de búsqueda para realizar consultas eficientes sin la necesidad de hacer uniones (joins).

    11. Colecciones en lugar de Tablas

    12. En lugar de tablas, MongoDB usa colecciones, donde se almacenan múltiples documentos.

    13. Las colecciones pueden almacenar documentos con estructuras variables.

    14. Base de Datos Distribuida

    15. MongoDB puede trabajar en un entorno distribuido, permitiendo replicación de datos y alta disponibilidad.

    Ejemplo de una Base de Datos en MongoDB#

    Supongamos que tenemos una base de datos llamada empresaDB con dos colecciones:

    1. Colección empleados
    {
    "_id": ObjectId("507f1f77bcf86cd799439011"),
    "nombre": "Juan Pérez",
    "cargo": "Desarrollador",
    "edad": 30,
    "salario": 50000
}
    1. Colección proyectos
    {
    "_id": ObjectId("507f191e810c19729de860ea"),
    "nombre": "Sistema de Ventas",
    "responsable": "Juan Pérez",
    "presupuesto": 100000
}

    Como puedes ver:

    • Cada documento es un objeto JSON con campos personalizados.
    • No es necesario que todos los documentos tengan los mismos campos.
    • Los documentos pueden referenciarse entre sí (por ejemplo, responsable en proyectos puede coincidir con un nombre en empleados).

    Comparación con Bases de Datos Relacionales#

    Característica MongoDB (NoSQL) Bases Relacionales (SQL)
    Estructura Documentos JSON/BSON Tablas con filas/columnas
    Esquema Flexible, sin esquema fijo Estricto, con esquema definido
    Escalabilidad Horizontal (sharding) Vertical (más potencia al servidor)
    Relación entre datos Referencias y embebidos Claves foráneas y joins
    Rendimiento Optimizado para lectura/escritura rápida Más lento en grandes volúmenes con joins

    En resumen, MongoDB es ideal para aplicaciones modernas que requieren escalabilidad, flexibilidad y rendimiento en grandes volúmenes de datos.

    Descripción de la Funcionalidad de la Aplicación MongoDB CRUD#

    Esta aplicación implementa un CRUD (Crear, Leer, Actualizar y Eliminar) sobre una base de datos MongoDB, permitiendo la gestión de usuarios y empleados. A continuación, se detallan sus principales funciones:

    1. Creación de Bases de Datos y Colecciones#

    • Se generan dos bases de datos:
    • crud_database: Contiene la colección users para almacenar información de usuarios.
    • employees_database: Contiene la colección employees para almacenar información de empleados.
    • Cada base de datos inicia con un registro por defecto para evitar que estén vacías.

    2. Inserción de Datos#

    • Insertar Usuario: Permite añadir un único usuario especificando su nombre, edad y ciudad.
    • Insertar Múltiples Usuarios: Opción para agregar varios usuarios a la base de datos en una sola operación.

    3. Consulta de Usuarios#

    • Recupera y muestra todos los usuarios almacenados en la base de datos.

    4. Actualización de Datos#

    • Permite modificar la información de un usuario existente, específicamente su edad.

    5. Eliminación de Datos#

    • Eliminar Usuario: Borra un usuario específico de la base de datos basándose en su nombre.

    6. Fusión de Colecciones#

    • Se combinan las colecciones users (de crud_database) y employees (de employees_database) en una nueva base de datos denominada merged_database, dentro de la colección merged_data.

    7. Menú de Navegación#

    • La aplicación proporciona una interfaz de menú en línea de comandos, permitiendo al usuario seleccionar las diferentes funcionalidades de manera interactiva.

    Esta estructura facilita la gestión de usuarios y empleados en MongoDB de forma sencilla y eficiente. 🚀

    Glosario de Conceptos en la Aplicación MongoDB CRUD#

    A#

    • Actualizar Usuario: Modificar los datos de un usuario en la base de datos, generalmente la edad.
    • Aplicación CRUD: Sistema que permite realizar las operaciones básicas de una base de datos: Crear, Leer, Actualizar y Eliminar.

    B#

    • Base de Datos: Contenedor estructurado donde se almacenan datos organizados en colecciones.

    C#

    • Colección: Equivalente a una tabla en bases de datos relacionales, almacena documentos dentro de una base de datos en MongoDB.
    • Crear Base de Datos y Colecciones: Acción de generar nuevas bases de datos (crud_database y employees_database) con sus respectivas colecciones (users y employees).

    D#

    • Documento: Unidad básica de almacenamiento en MongoDB, equivalente a una fila en bases de datos relacionales, representado en formato JSON.
    • DB1 (crud_database): Base de datos principal que almacena información de usuarios.
    • DB2 (employees_database): Base de datos secundaria que almacena información de empleados.
    • DB3 (merged_database): Base de datos que almacena la fusión de crud_database y employees_database.

    E#

    • Eliminar Usuario: Acción de borrar un usuario específico de la base de datos basándose en su nombre.
    • Empleado: Persona almacenada en la colección employees, generalmente con datos como nombre, cargo y salario.

    F#

    • Fusión de Colecciones: Proceso de combinar los datos de users y employees en una nueva colección merged_data dentro de merged_database.

    I#

    • Insertar Usuario: Acción de agregar un nuevo usuario en la colección users.
    • Insertar Múltiples Usuarios: Proceso que permite agregar varios usuarios en una sola operación a la base de datos.

    L#

    • Leer Usuarios: Recuperar y mostrar todos los documentos almacenados en la colección users.

    M#

    • MongoDB: Base de datos NoSQL que almacena datos en documentos JSON en lugar de tablas relacionales.
    • Menú de Navegación: Interfaz interactiva en la terminal que permite a los usuarios seleccionar acciones dentro de la aplicación.

    Q#

    • Query: Consulta utilizada para buscar datos dentro de una colección.

    S#

    • Salir: Opción del menú que finaliza la ejecución de la aplicación.

    U#

    • Usuario: Individuo almacenado en la colección users, con atributos como nombre, edad y ciudad.
