    Apuntes – Bloque 4: Streaming ligero y proyecto integrador#

    Objetivos del bloque#

    • Entender cómo capturar y enviar frames desde Unity.
    • Implementar un sistema de streaming ligero entre clientes.
    • Reunir en un proyecto final todas las funcionalidades aprendidas:
    • Chat
    • Salas
    • Multimedia
    • Streaming

    🎥 1. Streaming ligero en Unity#

    ¿Qué es?#

    • Consiste en capturar imágenes del juego (o de una cámara) y enviarlas por red.
    • No es streaming de vídeo en tiempo real, pero permite compartir gameplay en formato de imágenes continuas.

    🧱 2. Captura de imagen (frame) en Unity#

    Texture2D CapturaPantalla(int width, int height) {
    Texture2D tex = new Texture2D(width, height, TextureFormat.RGB24, false);
    tex.ReadPixels(new Rect(0, 0, width, height), 0, 0);
    tex.Apply();
    return tex;
}

    Convertir a byte array:#

    byte[] bytes = tex.EncodeToJPG();

    Enviar por red:#

    • Comprimir a Base64 y enviarlo como string:
    string base64 = Convert.ToBase64String(bytes);
websocket.SendText(JsonUtility.ToJson(new MediaFrame { type = "frame", data = base64 }));

    🖼️ 3. Mostrar imagen recibida#

    Texture2D tex = new Texture2D(2, 2);
byte[] bytes = Convert.FromBase64String(base64);
tex.LoadImage(bytes);
render.texture = tex;

    4. Formato del mensaje#

    {
  "type": "frame",
  "data": "<base64_image_data>"
}

    🧪 5. Proyecto integrador final#

    Objetivo#

    • Crear una app completa que combine todo lo aprendido.

    Requisitos#

    1. Permitir que varios jugadores se registren.
    2. Chat entre jugadores en diferentes salas.
    3. Enviar imágenes y vídeos en el chat.
    4. Activar el modo “streaming” para transmitir gameplay (simulado).
    5. Opción para que otros jugadores vean el streaming.

    💡 Sugerencias de ampliación#

    • Implementar control de permisos: ¿quién puede hacer streaming?
    • Añadir texto encima del streaming (overlay).
    • Incluir reacciones o emojis sobre el stream.
    • Crear un “modo espectador” para solo visualizar.

    Consejos técnicos#

    • El streaming consume muchos datos. Usa imágenes pequeñas o baja resolución.
    • Puedes controlar el framerate de envío (1 imagen cada 0.5 o 1 segundos).
    • Usa InvokeRepeating o Coroutine para el envío periódico.

    🔚 Resultado esperado#

    Al finalizar este bloque, deberías tener una aplicación de red funcional en Unity que: - Soporta chat con salas. - Transmite imágenes o gameplay. - Muestra mensajes multimedia. - Permite interacción básica tipo red social gamer.
