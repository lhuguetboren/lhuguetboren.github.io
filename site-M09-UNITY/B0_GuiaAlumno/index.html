    Guía de actividades del alumno: Desarrollo de chat y streaming en Unity#

    Objetivo general#

    En este módulo trabajarás cómo conectar jugadores entre sí dentro de Unity usando sistemas de red. Aprenderás a crear un chat funcional, implementar salas de conversación, enviar contenido multimedia y simular el envío de video en tiempo real.

    Bloque 1: Chat básico con Unity Transport (UDP)#

    Duración: 3 horas

    Qué aprenderás:#

    • Cómo se conectan varios jugadores en un servidor.
    • Qué es UDP y cómo funciona Unity Transport.

    Actividades:#

    1. Crear un servidor UDP que acepte múltiples jugadores.
    2. Programar el registro de jugadores (nombre).
    3. Enviar y recibir mensajes desde Unity.
    4. Mostrar los mensajes en pantalla.

    Qué entregarás:#

    • Escena funcional con chat entre al menos 2 jugadores.

    Bloque 2: Introducción a WebSocket y chat en tiempo real#

    Duración: 3 horas

    Qué aprenderás:#

    • Qué es WebSocket y cómo usarlo desde Unity.
    • Cómo conectarte a un servidor Node.js desde Unity.

    Actividades:#

    1. Probar un servidor WebSocket con mensajes simples.
    2. Crear un cliente Unity que se conecte al servidor.
    3. Enviar mensajes de chat y ver la respuesta.
    4. Añadir mensajes de estado (conectado, desconectado).

    Qué entregarás:#

    • Cliente Unity conectado a un servidor WebSocket.

    💬 Bloque 3: Salas de chat y contenido multimedia#

    Duración: 3 horas

    Qué aprenderás:#

    • Cómo dividir el chat en salas.
    • Cómo enviar imágenes o vídeos dentro del chat.

    Actividades:#

    1. Usar comandos como /join sala1 para moverte entre salas.
    2. Ver solo los mensajes de tu sala actual.
    3. Enviar enlaces a imágenes o vídeos.
    4. Ver esos contenidos dentro de Unity.

    Qué entregarás:#

    • Escena con varias salas y soporte multimedia básico.

    🎮 Bloque 4: Streaming y proyecto final#

    Duración: 3 horas

    Qué aprenderás:#

    • Cómo capturar y enviar una imagen en Unity.
    • Cómo aplicar todo lo anterior en un proyecto personal.

    Actividades:#

    1. Capturar pantalla o cámara del jugador.
    2. Enviar la imagen por red a otro jugador.
    3. Ver esa imagen remotamente.
    4. Integrar chat, salas, multimedia y/o streaming en una demo final.

    Qué entregarás:#

    • Mini proyecto personal o en pareja integrando lo aprendido.

    Recomendaciones:#

    • Trabaja en pareja si es posible.
    • Prueba con varios clientes en local.
    • Si te bloqueas, pide ayuda al profesor.
    • Sé creativo: puedes usar estos sistemas para juegos, redes sociales o apps educativas.

    Recursos que usarás:#

    • Unity + paquete NativeWebSocket.
    • Código base del profesor.
    • Servidor WebSocket en Node.js (proporcionado).
