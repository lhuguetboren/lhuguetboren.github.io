    Apuntes – Bloque 3: Salas de chat y multimedia con WebSocket#

    Objetivos del bloque#

    • Implementar salas (rooms) en un servidor WebSocket.
    • Asignar usuarios a salas y gestionar cambios de sala.
    • Enviar y recibir mensajes solo entre miembros de una misma sala.
    • Añadir soporte para mensajes multimedia (imágenes y vídeos).
    • Renderizar multimedia en la interfaz de Unity.

    1. ¿Qué es una sala de chat?#

    • Una sala es un grupo de usuarios que comparten mensajes entre ellos.
    • El servidor mantiene un registro de qué usuario está en qué sala.
    • Las salas permiten conversaciones privadas o por temas.

    2. Estructura de salas en el servidor Node.js#

    const rooms = {
  "general": new Set(),
  "juegos": new Set()
};
    • Cada conexión (WebSocket) se puede añadir o quitar de una sala.
    • Los mensajes solo se reenvían a los miembros de la sala actual del emisor.

    🛠️ 3. Flujo típico de comandos#

    { "type": "register", "name": "Rocío" }
{ "type": "join", "room": "sala1" }
{ "type": "chat", "text": "Hola sala1!" }
    • register: establece el nombre del usuario.
    • join: mueve al usuario a otra sala.
    • chat: se reenvía solo a los miembros de la sala actual.

    🖼️ 4. Multimedia en WebSocket#

    Envío#

    • El cliente envía una URL de imagen o video. 
      { "type": "media", "url": "https://..." }

    Recepción#

    • El cliente recibe el mensaje y determina si mostrar texto o imagen/video.
    • Se pueden usar etiquetas <img> o VideoPlayer en Unity para renderizar.

    🎮 5. Lógica del cliente Unity#

    Detección de tipo de mensaje#

    if (msg.type == "chat") {
    MostrarTexto(msg.from + ": " + msg.text);
} else if (msg.type == "media") {
    MostrarImagenDesdeURL(msg.url);
}

    Mostrar imagen desde URL#

    IEnumerator CargarImagen(string url) {
    UnityWebRequest www = UnityWebRequestTexture.GetTexture(url);
    yield return www.SendWebRequest();
    if (!www.result.Equals(UnityWebRequest.Result.ConnectionError)) {
        Texture2D tex = DownloadHandlerTexture.GetContent(www);
        imagenUI.texture = tex;
    }
}

    🧪 6. Ejercicios#

    1. Crear una interfaz para cambiar de sala.
    2. Mostrar el nombre de la sala activa en pantalla.
    3. Implementar el comando /rooms para listar salas.
    4. Añadir soporte para mostrar imágenes desde enlaces.
    5. Permitir mensajes que contengan texto e imagen a la vez.

    💡 Consejo#

    • Puedes usar Set para representar usuarios únicos por sala.
    • Para que los enlaces sean seguros, valida que terminen en .jpg, .png, .mp4, etc.
