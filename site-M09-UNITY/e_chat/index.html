    Programar un chat multicliente con jugadores registrados en Unity Transport#

    Este documento detalla los pasos y estructuras necesarias para implementar un chat multicliente donde jugadores registrados pueden conectarse al servidor, enviar mensajes y recibir mensajes de otros jugadores.

    🔧 Qué implica programarlo (a detalle)#

    1. 🧱 Estructura base del servidor#

    1.1. Crear el servidor#

    Como en CrossPlayServerBehaviour.cs, necesitas:

    • Inicializar MultiNetworkDriver.
    • Aceptar conexiones entrantes.
    • Usar NativeList<NetworkConnection> para manejar múltiples conexiones.

    1.2. Guardar nombres de usuario#

    Debes asociar cada NetworkConnection a un nombre de jugador:

    Dictionary<NetworkConnection, FixedString64Bytes> playerNames;

    Este diccionario te permitirá saber quién envió un mensaje y reenviarlo incluyendo su nombre.

    1.3. Flujo del servidor#

    → Cliente se conecta
→ Servidor solicita nombre de usuario
→ Cliente envía nombre
→ Servidor almacena { conexión ↔ nombre }
→ Cliente puede ahora enviar mensajes
→ Servidor reenvía a todos excepto al emisor

    1.4. Protocolo de mensajes#

    Para identificar distintos tipos de mensajes (registro vs texto), puedes definir un byte de tipo al inicio del stream:

    enum MessageType : byte
{
    Register = 0,
    Chat = 1
}

    2. Lógica del Servidor#

    Al recibir un mensaje#

    1. Leer MessageType del DataStreamReader.
    2. Si es Register, leer nombre y guardar en el diccionario.
    3. Si es Chat:
    4. Leer el mensaje.
    5. Obtener el nombre del emisor desde el diccionario.
    6. Crear un nuevo mensaje incluyendo el nombre y reenviarlo a todos los demás.
    foreach (var conn in m_Connections)
{
    if (conn != senderConnection)
    {
        m_Driver.BeginSend(NetworkPipeline.Null, conn, out var writer);
        writer.WriteByte((byte)MessageType.Chat);
        writer.WriteFixedString64(senderName);
        writer.WriteFixedString512(message);
        m_Driver.EndSend(writer);
    }
}

    🎮 Lógica del Cliente#

    1. Envío del nombre al conectarse#

    Cuando el cliente detecta que la conexión fue exitosa:

    • Envía un mensaje Register con su nombre.
    writer.WriteByte((byte)MessageType.Register);
writer.WriteFixedString64(playerName);

    2. Envío de mensajes de chat#

    Cuando el jugador escribe algo:

    • Enviar un mensaje Chat con el texto.
    writer.WriteByte((byte)MessageType.Chat);
writer.WriteFixedString512(message);

    3. Recepción de mensajes#

    Cuando recibe un mensaje:

    • Leer el MessageType.
    • Si es Chat, leer el nombre del emisor y el texto.
    • Mostrarlo en la UI del chat.

    📦 Requisitos adicionales#

    ✅ UI básica (para el cliente)#

    • Un input para escribir mensajes.
    • Un log o panel para mostrar mensajes entrantes.

    🛡️ Manejo de errores#

    • Verificar si la conexión sigue activa.
    • Manejar desconexiones y limpiar datos del servidor.
    • Validar que el nombre no esté vacío o duplicado.

    🧪 Ejemplo de mensaje enviado (por cliente)#

    [Byte: MessageType.Chat][FixedString512: "Hola a todos"]

    🧪 Ejemplo reenviado (por servidor)#

    [Byte: MessageType.Chat][FixedString64: "Juan"][FixedString512: "Hola a todos"]
