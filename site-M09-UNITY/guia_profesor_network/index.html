    Guia profesor network

    Guía didáctica para el profesor: Desarrollo de chat y streaming en Unity (UDP y WebSocket)#

    Objetivo general del módulo (12 horas)#

    Capacitar al alumnado en el desarrollo de sistemas de comunicación en red dentro de Unity, usando UDP (Unity Transport) y WebSocket. Se pretende que los alumnos creen un sistema de chat funcional con soporte para multimedia, salas de conversación y transmisión ligera de contenido.

    Bloque 1 (3h): Chat multicliente básico con Unity Transport (UDP)#

    Objetivos específicos:#

    • Entender el funcionamiento de la comunicación cliente-servidor.
    • Implementar un sistema de chat básico con Unity Transport.

    Contenidos:#

    • Introducción a UDP y Unity Transport.
    • Conexión multicliente.
    • Registro de usuarios.
    • Envío y recepción de mensajes.

    Actividades:#

    1. Explicación inicial (30 min):
    2. Introducir los conceptos básicos de red.

    3. Presentar la estructura general del chat UDP.

    4. Desarrollo guiado (2h):

    5. Configuración del servidor (MultiNetworkDriver).
    6. Manejo de conexiones y almacenamiento de nombres.
    7. Implementación de mensajes tipo Register y Chat.

    8. UI básica en Unity (input + log).

    9. Depuración y testeo (30 min):

    10. Pruebas con varios clientes.
    11. Gestión de errores.

    Recursos necesarios:#

    • Código base (extraído del documento proporcionado).
    • Plantilla de UI.

    Bloque 2 (3h): Introducción a WebSocket y chat simple#

    Objetivos específicos:#

    • Comprender el uso de WebSocket como alternativa a UDP.
    • Crear una conexión WebSocket desde Unity.

    Contenidos:#

    • ¿Qué es WebSocket?
    • Diferencias con UDP.
    • Librerías: NativeWebSocket, ws (Node.js).

    Actividades:#

    1. Exposición teórica (30 min):

    2. Explicación de los protocolos y librerías necesarias.

    3. Actividad práctica (1.5h):

    4. Montaje de servidor WebSocket en Node.js.

    5. Cliente Unity que se conecta, envía y recibe mensajes.

    6. Manejo de errores (1h):

    7. Reintentos de conexión.
    8. Indicadores de estado.

    Recursos:#

    • Servidor base en Node.js.
    • Paquete NativeWebSocket para Unity.

    Bloque 3 (3h): Salas de chat y multimedia (WebSocket)#

    Objetivos específicos:#

    • Implementar multicanales (salas).
    • Añadir soporte para multimedia (imágenes/videos).

    Contenidos:#

    • Estructura lógica de salas.
    • Comandos tipo IRC (/join, /leave).
    • Reproducción de imágenes y vídeos.

    Actividades:#

    1. Salas de chat (1.5h):
    2. Modificar el servidor para gestionar salas.
    3. Comandos de entrada/salida.

    4. Cliente Unity que lista y permite cambiar de sala.

    5. Multimedia (1.5h):

    6. Envío de enlaces.
    7. Carga y renderizado de imágenes/videos.
    8. Validaciones básicas.

    Recursos:#

    • Plantilla de servidor con soporte para salas.
    • UI de chat enriquecida.

    Bloque 4 (3h): Streaming y proyecto integrador#

    Objetivos específicos:#

    • Implementar transmisión de contenido por red.
    • Integrar todos los conocimientos en un mini proyecto.

    Contenidos:#

    • Captura de imagen/cámara en Unity.
    • Envío simulado por UDP.
    • Unificación de sistemas en una experiencia conjunta.

    Actividades:#

    1. Streaming ligero (UDP) (1.5h):
    2. Captura de pantalla.
    3. Conversión a Texture2D y bytes.

    4. Envío por red y visualización en otro cliente.

    5. Mini proyecto final (1.5h):

    6. Integración libre de chat, salas, multimedia y/o streaming.
    7. Presentación por grupos o individual.

    Evaluación:#

    • Funcionamiento del sistema.
    • Claridad en la interfaz.
    • Participación y originalidad.

    Observaciones generales:#

    • La dificultad es progresiva.
    • Se recomienda trabajo en parejas.
    • Puede adaptarse a proyectos personales posteriores.
