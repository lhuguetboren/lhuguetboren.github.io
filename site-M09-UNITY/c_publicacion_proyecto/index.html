    Proyecto: Publicación de Juegos en la Web con Base de Datos Compartida#

    Objetivo del Proyecto

    El propósito de este proyecto es que cada alumno cree su propio juego en Unity, lo publique en la web y lo conecte a una base de datos común donde se registrarán los jugadores y sus puntuaciones.

    Cada juego estará disponible en GitHub Pages y permitirá:

    ✔ Registrar jugadores en una base de datos central.

    ✔ Permitir que nuevos jugadores se den de alta desde cualquier juego.

    ✔ Guardar las estadísticas de partidas y puntuaciones.

    ✔ Consultar un ranking global de jugadores.

    ⸻

    Creación del Juego en Unity#

    Un sistema de puntuaciones. Un formulario para ingresar el nombre del jugador. Un botón para enviar la puntuación a la base de datos. Una pantalla de ranking donde se muestren las mejores puntuaciones.

    ⸻

    Exportación a WebGL y Publicación en GitHub Pages#

    ¿Por qué WebGL?

    • Nos permite jugar directamente desde el navegador sin instalar nada.
    • Podemos compartir nuestros juegos fácilmente.
    • Es ideal para demostrar proyectos en línea.

    Seleccionar WebGL como plataforma de compilación#

    1. Abre Unity y ve a File > Build Settings.

    2. En la ventana de Build Settings, selecciona WebGL.

    3. Si WebGL no está en la lista, presiona “Install with Unity Hub” para instalarlo.

    4. Una vez seleccionado, haz clic en “Switch Platform” (puede tardar unos minutos).

    Desactivar la compresión#

    1. Ve a Edit > Project Settings.

    2. En la pestaña izquierda, selecciona Player.

    3. Expande la sección Publishing Settings.

    4. Busca la opción Compression Format.

    5. Selecciona Disabled en lugar de Brotli o Gzip.

    Generar la compilación#

    1. Vuelve a File > Build Settings.
    2. Asegúrate de que WebGL sigue seleccionado.
    3. Haz clic en Build (elige una carpeta para guardar los archivos).

    ⸻

    Publicar el juego en github#

    Para publicar un juego Unity WebGL en GitHub Pages, sigue estos pasos:

    Hacer el Build en Unity#

    1. Ve a File > Build Settings y selecciona WebGL.

    2. En Player Settings > Publishing Settings, desactiva la compresión (Compression Format: Disabled).

    3. Haz clic en Build y guarda la salida en una carpeta (Build).

    Crear el repositorio en GitHub#

    1. Ve a GitHub y crea un nuevo repositorio.

    2. No añadas archivos como README.md o .gitignore por ahora.

    Subir los archivos a GitHub#

    1. Abre la carpeta del Build y crea un archivo index.html con este contenido:
    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Juego WebGL</title>
</head>
<body>
    <script>
        window.location.href = "Build/index.html";
    </script>
</body>
</html>
    1. Abre la terminal en la carpeta del Build y ejecuta:
    git init
git add .
git commit -m "Publicación WebGL"
git branch -M main
git remote add origin https://github.com/TU-USUARIO/TU-REPO.git
git push -u origin main

    4️⃣ Activar GitHub Pages

    1. En GitHub, ve a Settings > Pages.

    2. En Branch, selecciona main y la carpeta root.

    3. Guarda los cambios y espera unos minutos.

    Tu juego estará disponible en:

    https://TU-USUARIO.github.io/TU-REPO/

    Creación de la Base de Datos#

    Todos los juegos estarán conectados a una base de datos compartida donde se guardarán:

    • Los jugadores registrados (nombre y correo).

    • Las partidas jugadas (quién jugó, puntuación y fecha).

    • Un ranking global con los mejores jugadores.

    La base de datos estará en un servidor externo y los juegos se comunicarán con ella a través de una API (Interfaz de Programación de Aplicaciones).

    ⸻

    Conexión del Juego con la Base de Datos#

    Cada juego se conectará a la base de datos a través de peticiones HTTP:

    • Para registrar un jugador:

    • El jugador ingresa su nombre y correo.

    • Se guarda en la base de datos central.

    • Para guardar una puntuación:

    • Al terminar una partida, el juego envía la puntuación a la base de datos.

    • Para consultar el ranking:

    • El juego solicita la lista de los mejores jugadores y la muestra en pantalla.

    Ejemplo de flujo de datos:

    1. El jugador inicia el juego y se registra.

    2. El jugador juega una partida y obtiene una puntuación.

    3. El juego envía la puntuación al servidor.

    4. El juego muestra el ranking actualizado con los mejores jugadores.

    ⸻

    Consulta de Puntuaciones y Estadísticas#

    Cada juego incluirá un ranking global donde los jugadores podrán ver:

    • Las mejores puntuaciones del día.

    • El top 10 de jugadores.

    • Sus propias estadísticas de juego.

    Esto permitirá que los alumnos analicen datos y comparen sus puntuaciones con otros compañeros.

    ⸻

    Beneficios del Proyecto#

    ✔ Publicación en la web: Cada alumno tendrá su propio juego accesible desde cualquier dispositivo.

    ✔ Trabajo con bases de datos: Aprenderán cómo los juegos guardan información en servidores.

    ✔ Conexión con APIs: Experimentarán con el envío y recepción de datos entre Unity y un servidor.

    ✔ Competencia amistosa: Los alumnos podrán ver sus rankings y competir de manera saludable.

    ✔ Desarrollo de habilidades clave: Diseño de juegos, integración con la web y manejo de datos.

    ⸻

    Conclusión#

    Al finalizar este proyecto, cada alumno tendrá un juego publicado en la web y conectado a una base de datos real, permitiendo una experiencia más interactiva y competitiva. ¡Es un gran paso para aprender sobre juegos online y sistemas de datos en la nube!
